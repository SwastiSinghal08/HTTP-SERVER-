Mini HTTP Server

A TCP socket–based static file server built with raw C and Linux APIs. If you can see this page, your server is working perfectly.

Port 8080
Protocol HTTP/1.0
Method GET only
Model Single-threaded
How the server works
01 / SOCKET
Create TCP Socket
socket(AF_INET, SOCK_STREAM, 0) creates a file descriptor. Like dialling a phone — you get a "line" before talking.
02 / BIND
Bind to Port 8080
bind() assigns the socket to a local IP + port. The server claims "I live at :8080" so the OS knows where to deliver packets.
03 / LISTEN
Start Listening
listen() switches the socket to passive mode. It watches for incoming connections but doesn't accept them yet.
04 / ACCEPT
Accept a Client
accept() BLOCKS until a browser connects. It returns a NEW socket just for that client; the original keeps listening.
05 / PARSE
Parse HTTP Request
recv() reads the raw text. sscanf() extracts the method (GET) and path (/index.html) from the first line.
06 / RESPOND
Send HTTP Response
We open the file, read it with fread(), build headers, then send() them followed by the file bytes. Done!

Key concepts
IMP FUNCTIONS
socket()  → Create an endpoint for communication (returns fd)
bind()    → Assign a local address (IP + port) to the socket
listen()  → Mark socket as passive (ready to accept)
accept()  → Block until a client connects; return client socket
recv()    → Read data from a socket into a buffer
send()    → Write data from a buffer into a socket
close()   → Release the socket file descriptor
fopen()   → Open a file; "rb" = read binary mode
fread()   → Read N bytes from a FILE* into a buffer
malloc()  → Allocate heap memory; always pair with free()
htons()   → Host-to-Network Short: convert port to big-endian
INADDR_ANY → Bind to all available network interfaces
HTTP/1.0  → Stateless; connection closes after each response
MIME type → Tells browser how to handle the response body
